# 时序异常检测：难点分析、实验瓶颈与研究方向

> 更新：2026-03-20

---

## 一、时序异常检测的核心难点

### 1.1 标签稀缺问题（根本困难）

| 层面 | 具体表现 | 影响 |
|------|---------|------|
| **无全局 GT** | 436 个点位无标准异常标签 | 无法用传统 P/R/F1 评估 |
| **标签主观性** | 5 位评审的 Kappa 仅 0.18-0.35 | "什么是异常"本身有分歧 |
| **标签粒度粗** | 0/0.5/1 三档打分，不是逐点标注 | 无法精确评估检测边界 |
| **标签覆盖少** | 仅 100/436 = 23% 点位有评分 | 77% 数据完全无监督 |

**这是工业 TSAD 的普遍困境**——获取高质量标注的成本极高（需要领域专家逐点位审阅数小时的时序数据）。

### 1.2 异常形态多样性

| 异常类型 | 特征 | 检测难度 | 当前最优方法 |
|---------|------|---------|------------|
| 点异常（尖峰） | 单点或极短时间的极端值 | 低 | 3-Sigma, IForest |
| 阶跃（Level Shift） | 均值突然跳变 | 中 | Timer, OpenCV |
| 渐变漂移（Drift） | 缓慢偏离正常范围 | 高 | Timer |
| 方差突变 | 波动幅度突然改变 | 高 | Timer, MAD |
| 传感器卡死 | 长时间恒定值 | 中 | 简单规则即可 |
| 周期性异常 | 正常周期中嵌入异常周期 | 极高 | 无（当前所有方法均差） |
| 上下文异常 | 值本身不异常，但在特定上下文中异常 | 极高 | 需要多变量关联 |

**没有任何单一方法能覆盖所有异常类型。** Timer 在阶跃/漂移上表现好，但对上下文异常无能为力。

### 1.3 数据特征挑战

| 挑战 | 表现 | 影响 |
|------|------|------|
| **采样率不一致** | 1-40 秒/点，不同传感器差异大 | 统一降采样会损失信息 |
| **数据量巨大** | 46GB，单点位最大 170MB | 无法全量加载内存 |
| **传感器类型差异** | 14 种类型，异常模式完全不同 | 单一模型难以泛化 |
| **正负样本极不平衡** | 异常率 0.03%-27.5% | 模型倾向于全标为正常 |
| **缺失值和噪声** | 工业现场数据质量不稳定 | 预处理不当会引入伪异常 |

### 1.4 评估体系困境

当前自动评分器的已知缺陷：
- **对 ChatTS 偏差大**（预测 0.542 vs 实际 0.336）——基于规则的评分无法捕捉"视觉合理性"
- **对所有算法用同一规则**——偏向特定检测风格（中等异常率 + 多簇）
- **不区分异常类型**——某算法擅长尖峰但漏检漂移，和全面平庸的算法可能得分相同
- **无法评估检测延迟**——Timer 有 256 步前视窗口延迟，但评分不反映

## 二、当前实验的局限性与瓶颈

### 2.1 数据角度

| 瓶颈 | 现状 | 理想状态 |
|------|------|---------|
| **标注质量** | ADTK+HBOS 评分仅 0.586 作为训练标注 | 需要 Timer 级别（0.764）或人工逐点标注 |
| **标注数量** | 100 个评分点位，且仅三档打分 | 需要 200+ 点位的逐时间步二值标注 |
| **测试集偏小** | 99 个点位，某些传感器类型仅 3-4 个 | 各类型至少 20+ 个点位 |
| **无时间步级 GT** | 评分是"整体好不好"而非"哪里检测对了" | 需要逐段标注正确/错误 |
| **训练集无标注** | 337 个训练点位只有 ADTK 伪标签 | 需要更高质量的弱标签 |

### 2.2 算法角度

| 瓶颈 | 现状 | 理想状态 |
|------|------|---------|
| **Timer 是唯一有效的 TSFM** | Sundial 结果相同，MOMENT 过于保守 | 需要更多有效的时序基础模型 |
| **单变量检测** | 所有方法独立处理每个传感器 | 多变量关联检测 |
| **离线全量推理** | 需要全部数据才能检测 | 在线流式检测 |
| **无异常类型识别** | Timer 只输出 mask，不区分类型 | 需要类型标签（如阶跃/漂移/尖峰） |
| **降采样信息损失** | M4 降到 10000 点丢失细节 | 自适应降采样或多尺度检测 |
| **图像化方法失效** | Qwen-VL/GAF 均不如数值方法 | 需要更好的时序→视觉编码 |

### 2.3 工程角度

| 瓶颈 | 现状 | 理想状态 |
|------|------|---------|
| **GPU 资源受限** | 2×48GB，经常被其他用户占用 | 专用 GPU 或队列调度 |
| **依赖项目 2** | Timer/ChatTS 适配器依赖 ts-iteration-loop | 应抽取独立 |
| **环境碎片化** | 7 个 conda 环境 | 统一管理或容器化 |

## 三、研究调研方向（面向论文）

基于上述分析，提出以下可发表的研究方向：

### 方向 1：无标签/弱标签工业时序异常检测

**核心问题**：如何在没有 Ground Truth 的情况下训练和评估异常检测模型？

**调研关键词**：
- Unsupervised time series anomaly detection benchmark
- Weakly supervised anomaly detection
- Self-supervised anomaly detection in time series
- Active learning for anomaly labeling
- Pseudo-label bootstrapping for TSAD

**潜在创新点**：
- 用 Timer 等强模型生成伪标签，逐步迭代提升标注质量（Teacher-Student）
- 设计不依赖 GT 的评估指标（基于物理约束的自动评估）
- 多评审人的一致性加权评估框架

### 方向 2：多尺度/多粒度异常检测

**核心问题**：如何同时检测点异常（毫秒级）和趋势漂移（小时级）？

**调研关键词**：
- Multi-scale time series anomaly detection
- Hierarchical temporal anomaly detection
- Wavelet-based anomaly detection
- Multi-resolution time series analysis

**潜在创新点**：
- 多尺度降采样 + 分层 Timer 推理（粗→细）
- 小波分解后分别检测不同频率分量的异常
- 注意力机制在不同时间尺度上的异常聚焦

### 方向 3：传感器类型感知的自适应检测

**核心问题**：如何根据传感器特性自动调整检测策略？

**调研关键词**：
- Sensor-type-aware anomaly detection
- Meta-learning for TSAD
- Adaptive threshold selection
- Domain-specific time series foundation models

**潜在创新点**：
- 传感器类型自动识别 + 策略路由（本项目已有分层策略基础）
- 元学习框架：快速适应新传感器类型
- 异常率先验驱动的自适应阈值

### 方向 4：时序异常检测的可解释性

**核心问题**：Timer 输出 mask 但不解释"为什么"——如何生成人类可理解的异常解释？

**调研关键词**：
- Explainable anomaly detection
- LLM-based anomaly explanation
- Attention-based anomaly attribution
- ChatTS time series analysis

**潜在创新点**：
- Timer 检测 + ChatTS/LLM 解释（两阶段）
- 注意力可视化定位异常根因
- 自然语言异常报告生成

### 方向 5：多变量关联异常检测

**核心问题**：当前所有方法都是单变量——如何利用传感器间的物理关联？

**调研关键词**：
- Multivariate time series anomaly detection
- Graph neural network for TSAD
- Spatial-temporal anomaly detection
- Inter-sensor correlation anomaly

**潜在创新点**：
- 构建传感器关联图（基于相关性/因果性）
- GNN 学习正常关联模式，偏离 = 异常
- 同类设备的对比检测（同型号传感器互相参照）

### 方向 6：时序→视觉的有效编码（当前实验的教训）

**核心问题**：Qwen-VL 和 GAF 为什么失败？如何设计保留异常信号的视觉编码？

**调研关键词**：
- Time series to image encoding for anomaly detection
- Recurrence Plot anomaly detection
- Gramian Angular Field vs Markov Transition Field
- Visual anomaly detection in time series

**潜在创新点**：
- 对比不同编码方式（GAF/MTF/RP）对异常保留度的影响
- 设计"异常敏感"的时序→图像编码（放大异常区域的视觉差异）
- 多通道编码：原始值 + 一阶差分 + 二阶差分 → RGB 三通道
