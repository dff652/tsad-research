## 一、实验背景与目标

### 1.1 业务场景

工业传感器（温度、压力、流量、电流等）持续产生高频时序数据。需要从 **436 个点位**的数据中自动识别异常（设备故障、传感器失效、过程偏移等），替代人工巡检。

### 1.2 数据现状

| 项 | 数量 | 说明 |
|----|------|------|
| 总点位 | 436 | ADTK+HBOS 已推理，CSV 共 46GB |
| 有人工评分 | 100（实际 99 有 CSV） | 5 位专家 × 3 个模型 × 0/0.5/1 评分 |
| 传感器类型 | 14 种 | Temperature 最多(100)，Speed 最少(1) |
| 时间跨度 | 2023-07 ~ 2024-11 | 约 16 个月 |
| 采样频率 | 1-40 秒/点 | 不同传感器差异大 |

### 1.3 核心挑战

1. **无全局 Ground Truth**：436 个点位没有标准异常标签，只有 100 个点位的主观打分
2. **评审人分歧大**：Fleiss' Kappa 仅 0.18-0.35（弱一致性）
3. **异常形态多样**：点异常、阶跃、漂移、方差突变等无法用单一方法覆盖
4. **数据量巨大**：46GB 原始 CSV，逐个读取不可行

### 1.4 实验目标

- 在统一框架下对比 **6 种检测范式、20+ 种方法/配置**
- 找到工业场景下的 SOTA 方案
- 评估"时序转图像→目标检测"新范式的可行性

## 二、评估体系设计

### 2.1 为什么不能用传统的 Precision/Recall

传统异常检测评估依赖 Ground Truth 标签。本项目面临：
- 436 个点位中 **336 个没有任何标签**
- 100 个评分点位的标签是 **主观打分（0/0.5/1）而非二值标注**
- 评审人之间一致性弱（Kappa 0.18-0.35），说明"什么是异常"本身就有争议

因此设计了 **双维度评估体系**：

### 2.2 维度 A：主观经验拟合（100 评分点）

**目的**：评估算法是否"符合工业专家感觉"

**指标**：
- `自动预测评分`：基于算法输出特征（异常率、簇数量等）预测人工可能给出的评分
- `通过率`：预测评分 ≥ 0.5 的点位比例
- `良好率`：预测评分 ≥ 0.8 的点位比例

**评分规则**（基于 Timer 人工评分的统计规律反推）：
| 规则 | 评分 | 原因 |
|------|------|------|
| 异常率 = 0（漏检） | 0.2 | Timer 低分组的 ADTK 异常率仅 0.97% |
| 异常率 > 50%（乱叫） | 0.1 | 全图标红 = 没有检测价值 |
| 异常率 0.1%-15% | 0.65 基础分 | Timer 高分组的异常率区间 |
| 簇数量 > 50 | +0.10 | 精细检测 = 高分特征 |
| 适中簇长度 10-5000 | +0.05 | 合理区间长度 |

**选择理由**：自动评分器在 Timer 上的偏差仅 0.092（预测 0.764 vs 实际 0.672），具有参考价值。虽然不完美，但在无 GT 的情况下是最可行的量化方案。

**已知局限**：
- 评分器基于规则而非学习，无法捕捉"视觉合理性"
- 对所有算法使用同一规则，可能偏向某种检测风格
- ChatTS 预测 0.542 vs 实际 0.336，偏差较大（0.206）

### 2.3 维度 B：物理常识约束（436 全量点）

**目的**：过滤"乱叫狂响"的不稳定模型

**约束**：
| 约束 | 阈值 | 原因 |
|------|------|------|
| 平均异常率上限 | 15% | 超过则检测价值为负 |
| 平均异常率下限 | 0.1% | 低于则等于没检测 |
| 跳变率上限 | 30% | 异常标记应连续，频繁 0/1 切换不合理 |

**选择理由**：工业传感器数据有物理规律——真实异常通常是连续的时间段而非随机散点。这些约束不依赖标签，纯粹基于物理常识。

### 2.4 维度 C：Grounding 评估（Qwen-VL 目标检测专用）

**指标**：
| 指标 | 定义 | 用途 |
|------|------|------|
| Precision | TP / 预测框数 | 预测的准确性 |
| Recall | TP / GT框数 | 是否覆盖了真实异常 |
| F1 | 2PR/(P+R) | 综合指标 |
| IoU | 预测框与GT框的交并比 | bbox 定位精度 |

**匹配规则**：1D IoU（仅 x 方向）> 0.1 视为匹配，贪心匹配（按 IoU 降序）

### 2.5 评估体系的不足与改进方向

| 不足 | 影响 | 改进方向 |
|------|------|---------|
| 自动评分基于规则 | 无法衡量"视觉合理性" | 引入更多评审人标注 |
| 仅对比异常率级别 | 不区分"正确标记"和"碰巧在合理区间" | 需要逐点位的标注对比 |
| 不区分异常类型 | 跳变和漂移的检测难度不同 | 按异常类型分层评估 |
| 训练/测试集基于 ADTK 标注 | ADTK 自身评分仅 0.586 | 用 Timer 标注替代 |

## 三、实验设计

### 3.1 数据划分

| 集合 | 用途 | 点位数 | 文件 |
|------|------|--------|------|
| 测试集 | 仅评估，不训练 | 99 | `data/cleaned/evaluated_points.txt` |
| 训练集 | 微调/适配 | 337 | `data/cleaned/train_points.txt` |

**严格规则**：100 个有人工评分的点位绝不用于训练。

### 3.2 标准化流程

所有算法通过统一流水线评测：
```
原始 CSV (46GB) → 降采样 → 算法推理（subprocess 隔离）→ 标准 mask → 评估
```

- 降采样：M4 方法到 5000-10000 点（保留极值特征）
- 隔离：每个算法在独立 conda 环境中，通过 subprocess 调用
- 输出：标准 CSV 格式 `global_mask` 列（0=正常，1=异常）

### 3.3 环境隔离

| 算法 | conda 环境 | 关键依赖 |
|------|-----------|---------|
| Timer/Sundial | timer | torch 2.6, transformers 4.40 |
| ChatTS | chatts | torch 2.6, transformers 4.52 |
| Qwen-VL | qwen_tune | torch 2.10, transformers 4.57 |
| Qwen3.5 | qwen35 | torch 2.10, transformers 5.3 |
| MOMENT | moment | torch 2.10, momentfm 0.1.4 |
| 统计方法 | base | sklearn, scipy |

**关键发现**：用户级 `~/.local/` 的 transformers 4.52 会覆盖 conda 内版本，导致 Timer 推理报错。解决方案：`PYTHONNOUSERSITE=1`。

## 四、算法逐一分析

### 4.1 Timer-84M（SOTA）

| 维度 | 值 |
|------|---|
| 范式 | 预测→残差→MAD 阈值 |
| 预测评分 | **0.764** |
| 实际人工分 | **0.672** |
| 通过率 / 良好率 | 94.9% / 87.9% |
| 异常率 | mean=3.28%, median=2.16% |
| 推理速度 | ~60s/点位 |
| 零异常点位 | 0 |
| GPU 需求 | ~6GB |

**原理**：清华大学 Timer 模型（84M 参数），基于 Transformer 的时序预测。将原始时序降采样到 10000 点，用 256 长度的滑动窗口逐点预测下一步值，计算残差。残差超过 MAD 3.5 倍 = 异常。

**优点**：
- 评分遥遥领先所有其他方法（第二名 IForest 仅 0.670）
- 检测精细：平均 599 个短簇（139 点/簇），精确定位异常区间
- 零漏检：每个点位都检测到异常
- 物理合理：异常率 3.3% 在合理范围

**缺点**：
- 速度慢（60s/点，436 全量需 7 小时）
- 依赖 GPU
- 在"平静"数据上可能过度检测（低分点位的簇数量反而更多 670 vs 500）

**关键分析**：
- Timer 推理特征与人工评分直接相关性弱（max r=-0.24），说明人工评分不依赖简单统计
- Timer 高分场景 = 数据本身有明显异常 + Timer 检测更精炼（簇少但准）
- Timer 与 ADTK 异常率高度相关（r=0.69），但粒度完全不同

### 4.2 Sundial-128M

| 维度 | 值 |
|------|---|
| 范式 | 同 Timer |
| 预测评分 | 0.725（仅 4 点位） |

**关键发现**：在标准流程下（M4 降采样 10000 点 + MAD k=3.5），Sundial 与 Timer **产生 100% 相同的输出**。对比了 AI_20405E.PV 的 3,733,620 个数据点，差异为 0。

**原因**：降采样抹平了模型间的细微预测差异，MAD 阈值的离散化进一步消除残差区别。

**结论**：要区分两者，需更精细的降采样或连续评分而非二值 mask。

### 4.3 ChatTS-14B

| 维度 | 值 |
|------|---|
| 范式 | LLM 文本理解时序 |
| 预测评分 | 0.542 |
| 实际人工分 | 0.336 |
| 通过率 / 良好率 | 73.7% / 0.0% |
| 异常率 | mean=1.09%, median=0.76% |
| 推理速度 | ~11s/点位 |
| 零异常点位 | **24** |

**原理**：字节跳动 ChatTS-14B（4-bit 量化），将降采样后的时序（768 点）输入 LLM，用 prompt 引导模型输出 JSON 格式的异常区间。

**优点**：
- 推理快（11s，Timer 的 1/6）
- 能输出异常类型描述（如"设备故障"、"传感器失效"）
- 4-bit 量化仅需 ~10GB 显存

**缺点**：
- **漏检严重**：24/99 点位零异常（Timer 为 0）
- **JSON 解析不稳定**：LLM 输出的 JSON 经常格式错误（中文括号、缺逗号）
- 良好率 0%：所有点位评分都在"中等"档，没有"优秀"
- 自动评分（0.542）远高于实际人工分（0.336），说明评分器对 ChatTS 过于宽容

**JSON 解析问题处理**：在适配器中实现了容错解析器（正则逐个提取 `{...}` 对象），将解析成功率从约 60% 提升到 95%+。

### 4.4 Qwen-VL-8B（旧版，业务平台已有）

| 维度 | 值 |
|------|---|
| 范式 | VLM 看图识别异常 |
| 实际人工分 | 0.225 |
| 差评率 | 68.2% |

**原理**：将时序绘制为图片，用 Qwen-VL 视觉模型分析图中异常区域。

**原业务平台中的效果**：5 位专家评分中排名最后，68.2% 差评率。

### 4.5 Qwen3-VL-8B Grounding（新实验）

| 维度 | 零样本 | 微调后 |
|------|--------|--------|
| 预测评分 | 0.446 | 0.447 |
| 异常率 | 31.4% | 30.5% |
| Grounding Precision | 0.216 | 0.220 |
| Grounding Recall | 0.306 | 0.324 |
| Grounding F1 | 0.167 | 0.173 |
| Grounding IoU | 0.315 | 0.329 |
| TP | 74/968 | 86/968 |
| 推理速度 | ~10s/点 | ~14s/点 |

**原理**：将异常检测转化为图像目标检测——时序图中的异常区域 = 需要 bbox 框选的目标。使用 Qwen3-VL 的 grounding 能力，输出 `bbox_2d [x1,y1,x2,y2]`（0-1000 坐标）。

**微调细节**：
- 训练数据：337 训练集点位中 223 个有异常标注的图片
- 标注来源：ADTK+HBOS 的异常区间转换为 bbox 坐标
- LoRA：rank=16, alpha=32, 目标层 q/k/v/o_proj
- 训练：3 epoch, loss 10.4→3.0
- 图片缩至 800px + gradient checkpointing（原 4000×800 OOM）

**优点**：
- 零漏检（所有图都检测到异常）
- bbox 格式化输出，可直接可视化
- 微调后 TP 匹配 +16%（74→86），bbox 更精炼（685→590）

**缺点**：
- **严重过检**：异常率 31%（Timer 3.3%），22/99 点位输出覆盖全图的大框
- **bbox 过大**：模型倾向于输出少量大框而非多个精确小框
- **标签单一**：570/590 个 bbox 标签为泛化的 "anomaly"
- **微调改善有限**：F1 仅 +3.6%

**根因分析**：
1. VLM 缺乏时序领域知识——把正常周期波动也识别为"异常"
2. 像素精度不足——4000px 图中 1px ≈ 数百个数据点，无法精确定位边界
3. ADTK 标注质量差（评分 0.586）限制了微调上限

**Qwen3.5-27B 评估**：确认为多模态模型（有 vision_config），但 4-bit 量化仍需 ~46GB，48GB 单卡 OOM，当前硬件不可用。

### 4.6 MOMENT-1-large（CMU, ICML'24）

| 维度 | 值 |
|------|---|
| 范式 | TSFM 重建→残差 |
| 预测评分 | 0.500 |
| 通过率 / 良好率 | 100% / 0% |
| 异常率 | mean=0.03%, median=0.02% |
| 推理速度 | ~8.5s/点位 |

**原理**：T5-based 385M 参数通用时序基础模型。预训练了重建头，输入时序窗口（512 点）→重建→残差 = 异常分数。使用 95 百分位阈值判定异常。

**优点**：
- 通过率 100%（所有点位异常率在合理区间）
- 推理快，不需要滑动窗口逐点预测
- 零样本，无需微调

**缺点**：
- **过于保守**：异常率仅 0.03%（Timer 3.3% 的 1/100）
- 良好率 0%：检测到的异常太少，无法产生"优秀"的检测效果
- 论文自身 benchmark 中 K-NN 击败了 MOMENT 的异常检测

**根因分析**：MOMENT 的异常检测是预训练副产品而非核心能力。重建头重建能力太强（loss 收敛到很低），对异常窗口也能较好重建，导致异常/正常区分度不足。

### 4.7 IForest（Isolation Forest）

| 维度 | 值 |
|------|---|
| 范式 | 统计（基于数值分布） |
| 预测评分 | **0.670** |
| 通过率 / 良好率 | 77.8% / 50.5% |
| 异常率 | mean=0.17%, median=0.17% |
| 推理速度 | ~1.5s/点位 |

**原理**：孤立森林——通过随机切分特征空间，异常点更容易被"孤立"（路径更短）。设定 contamination=1%。

**优点**：
- **评分第 3（仅次于 Timer 和集成方法）**
- 极快（1.5s/点），无需 GPU
- 实现简单，sklearn 一行代码
- 良好率 50.5%（远高于 ChatTS 的 0%）

**缺点**：
- 异常率偏低（0.17%），可能漏检
- 不考虑时序上下文（每个点独立判断）
- 对阶跃/漂移等持续性异常不敏感

**重要发现**：简单统计方法在此任务上评分不低，印证了 TSB-AD benchmark 的结论——复杂模型不一定优于简单方法。

### 4.8 ADTK+HBOS（业务平台 Baseline）

| 维度 | 值 |
|------|---|
| 范式 | 统计（HBOS 直方图密度） |
| 预测评分 | 0.586 |
| 异常率 | mean=4.68%（436 全量），mean=7.65%（100 评分点） |
| 簇特性 | 21 个簇，平均 19356 点/簇 |

**优点**：
- 覆盖全量 436 点位的已有结果
- 通过物理约束检查

**缺点**：
- **过度粘连**：平均簇长度 19356 点（Timer 仅 139）——标记了超长连续区间
- 82 个零异常点位（Timer 为 0）
- Current/Power 类异常率高达 27.5%

### 4.9 3-Sigma / MAD / IQR

| 方法 | 评分 | 异常率 | 特点 |
|------|------|--------|------|
| 3-Sigma | 0.436 | 0.12% | 最保守，43 个零异常 |
| MAD | 0.498 | 0.99% | 比 3-Sigma 敏感 |
| IQR | 0.529 | 0.82% | 四分位距，35 个零异常 |

**共同缺点**：不考虑时序上下文，零异常点位多，对低幅度的持续异常不敏感。

### 4.10 集成方法

| 策略 | 评分 | 异常率 | 设计 |
|------|------|--------|------|
| **Weighted** | **0.717** | 5.09% | Timer 权重 3x + 统计方法 1x |
| Voting (≥2) | 0.617 | 0.84% | 至少 2 种方法同时标记 |
| Cascade | 0.443 | 0.23% | Timer 检测→统计方法过滤 |

**分析**：
- **Weighted 效果接近 Timer**（因为 Timer 权重最高，本质是 Timer+增强）
- **Cascade 过于保守**——统计方法异常率极低，过滤掉了 Timer 的大部分正确检测
- Timer 与统计方法检测的异常**重合度极低**（不同维度的异常），集成增益有限

### 4.11 OpenCV 边缘检测

| 维度 | 值 |
|------|---|
| 范式 | 计算机视觉（Canny + Hough 线检测） |
| 预测评分 | 0.488 |
| 通过率 | 91.9% |
| 推理速度 | **0.4s/点位**（最快） |

**原理**：将时序绘制为图片 → Canny 边缘检测 → HoughLinesP 检测垂直线段 → 垂直线 = 时序跳变点。

**优点**：极快，无需 GPU，无需训练
**缺点**：只能检测"跳变/阶跃"，无法检测漂移、方差变化等

### 4.12 GAF + ConvAutoencoder

| 维度 | 值 |
|------|---|
| 范式 | 时序→GAF 图像→重建→异常 |
| 预测评分 | 0.500 |
| 异常率 | 0.02% |
| 训练数据 | 46010 个 GAF 窗口（337 训练点位） |

**原理**：
1. Gramian Angular Field 将时序窗口编码为 2D 图像（保留角度关系）
2. 训练 ConvAutoencoder 重建正常 GAF 图像
3. 重建误差高 = 异常窗口

**实验迭代**：
- v1（128 channels bottleneck）：异常率 0.02%，max/mean 误差比仅 3.7x
- v2（4 channels 窄 bottleneck）：异常率仍 0.02%，loss 从 0.01 升到 0.13 但区分度不增

**失败原因分析**：
- GAF 是角度编码——局部数值跳变在 GAF 图中仅表现为小范围像素差异
- ConvAE 对异常和正常窗口的重建误差差异太小
- 文献报告的 F1≈99% 是在标准 benchmark 数据集上（SWaT 等），工业真实数据的异常模式更复杂

### 4.13 TransformerKL（自监督）

| 维度 | 值 |
|------|---|
| 范式 | 自监督（噪声先验训练 + 注意力 KL 散度） |
| 异常率 | 0.11%（单点测试） |
| 训练时间 | 609s/点位 |

**原理**：用粉红噪声训练 Transformer 重建模型，引入注意力 KL 散度正则化。在真实数据上，注意力偏离先验 = 异常。

**缺点**：每个点位独立训练（609 秒），无法批量使用。需改为全局预训练模型。

## 五、参数调优实验

### 5.1 Timer threshold_k 扫描

在 10 个代表性点位上（覆盖高/中/低评分和不同传感器类型）扫描 5 种阈值：

| threshold_k | 异常率 | 自动评分 |
|-------------|--------|---------|
| 2.5 | 9.66% | 0.760 |
| 3.0 | 7.89% | 0.760 |
| **3.5（默认）** | **6.88%** | **0.760** |

## 六、核心结论

### 6.1 范式效果排序

| 排名 | 范式 | 代表算法 | 最佳评分 | 结论 |
|------|------|---------|---------|------|
| **1** | **预测→残差** | **Timer-84M** | **0.764** | **工业 TSAD 最优范式** |
| 2 | 集成 | Weighted Ensemble | 0.717 | 本质是 Timer+增强 |
| 3 | 统计 | IForest | 0.670 | 简单有效，速度快 |
| 4 | LLM 文本 | ChatTS-14B | 0.542 | 漏检严重 |
| 5 | TSFM 重建 | MOMENT | 0.500 | 过于保守 |
| 6 | CV 边缘 | OpenCV | 0.488 | 仅检测跳变 |
| 7 | VLM 目标检测 | Qwen3-VL | 0.447 | 过检严重 |
| 8 | GAF+CNN | ConvAE | 0.500 | 区分度不足 |

### 6.2 关键发现

1. **Timer 的"预测→残差"范式是工业 TSAD 的最优选择**——它理解时序上下文，能区分正常波动和真实异常
2. **图像化方法全面不如数值方法**——无论是"看图"（Qwen-VL）还是"转图"（GAF），都在信息编码时损失了异常检测所需的数值精度
3. **简单统计方法（IForest）效果不差**——评分第 3，速度是 Timer 的 40 倍
4. **特征-评分相关性弱**（max r=0.24）——说明人工评分依赖"视觉合理性"等难以量化的维度
5. **评审人一致性低**（Kappa 0.18-0.35）——"什么是异常"本身就有主观性
6. **ChatTS 的 LLM 范式有独特价值**（能输出语义描述）但当前漏检严重

### 6.3 尚需考虑的维度

| 维度 | 当前状态 | 建议 |
|------|---------|------|
| **检测延迟** | 未评估 | 异常发生后多久能检测到？Timer 有 256 点前视窗口延迟 |
| **误报成本 vs 漏报成本** | 未区分 | 工业场景下漏报（设备损坏）成本远高于误报（多看一眼） |
| **在线推理** | 未测试 | 当前全离线，实际需要流式/增量检测 |
| **多变量关联** | 未考虑 | 单变量检测，忽略了传感器间的相关性 |
| **季节性/趋势** | 降采样可能抹平 | STL 分解后的残差可能更适合检测 |
| **标注一致性** | Kappa 仅 0.18-0.35 | 需要更严格的标注规范或使用主动学习 |
| **模型鲁棒性** | 未测试 | 对缺失值、采样率变化、传感器漂移的鲁棒性 |
| **计算成本** | 仅记录速度 | 需要 GPU 利用率、能耗的综合评估 |
| **可解释性** | ChatTS 有语义输出 | Timer 仅输出 mask，不解释"为什么是异常" |
| **长尾异常** | 未专门评估 | 罕见异常类型（如传感器卡死、通信中断）的检测能力 |

### 4.15 Teacher-Student 伪标签

| 维度 | 值 |
|------|---|
| 预测评分 | 0.471 |
| 异常率 | 0.04% |

**原理**：从 Timer 标记的异常/正常窗口中学习统计特征阈值（std, range, diff_max, kurtosis），用阈值检测新数据。

**失败原因**：异常窗口与正常窗口在统计特征上的分离度极低（max separation 仅 0.59）。Timer 的优势来自**时序上下文理解**（"这个值在当前趋势中是否合理"），不是简单的统计偏离。

### 4.16 TS2Vec 对比学习

| 维度 | 值 |
|------|---|
| 预测评分 | 0.450-0.500 |
| 训练 | CPU, 20 epoch, 50 点位 |

**原理**：层级化对比学习预训练 → 有/无 mask 两次编码 → 表示距离 = 异常分数。

**失败原因**：CPU 上 20 epoch 训练不充分，对比学习需要更长的训练和 GPU 加速。

### 4.17 SENSE 选择性集成

| 维度 | 值 |
|------|---|
| 预测评分 | 0.764 |
| 路由规则 | 高变异→Timer+Wavelet, 高峰度→IForest+MAD |

**结论**：评分与 Timer 持平。路由规则有效，但 Timer 适用于绝大多数传感器类型，路由最终几乎总是选择 Timer 为主。

### 4.18 小波多尺度 / 频率 Patching

| 方法 | 单独评分 | 与 Timer 融合 |
|------|---------|-------------|

| FreqPatch (FFT 8-band) | 0.500 | 0.763 |

**关键发现**：信号处理方法单独效果一般，但与 Timer 融合后**互补提升**——Timer 擅长趋势/上下文异常，小波/FFT 擅长高频尖峰/阶跃。并集融合实现了新 SOTA。
